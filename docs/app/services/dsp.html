Digital Signal Processing (DSP)¶

Module for digital signal processing related functions.

This module is needed for knowing the minimum time of recording per region for the Scan&Paint method and to actualy get the spectrum analysis for a single audio signal.

app.package.services.dsp.get_spectrum(audio: numpy.ndarray, fs: int, limits=[12, 20000]) → tuple¶

Function that returns one third-octave spectrum analysis of a signal

It does so while windowing the signal with a Hannin window of size 2^15 samples, with an overlap of 50%

Parameters

  • audio (np.ndarray) – Audio two-dimensional array. First dimension: number of channels. Second dimension: length of the audio

  • fs (int) – sampling rate

  • limits (list, optional) – Limits for the frequency

  • analysis. Defaults to None. (spectrum) –

Returns

Returns the spectrum values and the frequency array.

Return type

tuple[list, list]

app.package.services.dsp.get_time_of_recording(lowest_freq, e=0.1, frac=3, fs=48000)¶

Function that returns the minimum time of recording for a given frequency range.

There is an error related to analyzing random data signals, called the Normalized Random Error. If this error is less than 20 %, it can be approximated as a function of the bandwidth and the time of recording.

If we perform a one third-octave spectrum analysis over a signal, the maximum error will be directly related to the bandwidth of the lowest band.

Parameters

  • lowest_freq (float) – The lowest frequency considered for the analysis

  • e (float, optional) – Maximum error tolerated. Defaults to 0.1.

  • frac (int, optional) – Fraction for the 1/x octave spectrum analysis. Defaults to 3.

  • fs (int, optional) – Audio sampling frequency. Defaults to 48000.

Returns

Minimum time of recording per area to analyze.

Return type

float
